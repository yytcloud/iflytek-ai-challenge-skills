# 电商评论分析框架

## 分析维度定义

### 1. 商品质量（Weight: 30%）
- 子维度：材质/用料、做工精细度、功能完整性、耐用性、安全性
- 评分标准：
  - 优秀（90+）：大量用户夸赞质量好、用了很久没坏
  - 良好（70-89）：偶有质量小问题但总体满意
  - 一般（50-69）：质量评价两极分化
  - 较差（<50）：频繁出现质量问题投诉

### 2. 外观设计（Weight: 15%）
- 子维度：颜色准确度、款式颜值、尺寸合适度、整体质感
- 重点关注："色差"是高频投诉点

### 3. 性价比（Weight: 20%）
- 子维度：价格合理性、促销体验、与同类对比
- 信号词："值了"、"物超所值" vs "不值这个价"、"贵了"

### 4. 物流体验（Weight: 15%）
- 子维度：发货速度、运输时间、包装完好度、快递态度
- 注意：物流问题通常不是商家直接责任，但影响复购

### 5. 客户服务（Weight: 10%）
- 子维度：响应速度、服务态度、问题解决能力、退换货体验

### 6. 使用体验（Weight: 10%）
- 子维度：上手难易度、实际效果、场景适配性、持续使用意愿

## 情感强度分级

| 等级 | 标签 | 特征 |
|---|---|---|
| 5 | 极度正面 | 带感叹号、推荐词、回购承诺 |
| 4 | 正面 | 基本满意，无明显不满 |
| 3 | 中性 | 有好有坏、客观描述 |
| 2 | 负面 | 有明确不满、抱怨 |
| 1 | 极度负面 | 愤怒、要求退款、警告他人不要买 |

## 痛点严重度计算

严重度 = 出现频次 × 情感强度均值 × 业务影响系数

业务影响系数：
- 质量问题：1.5（直接影响产品口碑）
- 描述不符：1.3（影响转化和退换货成本）
- 物流问题：0.8（可改善）
- 服务问题：1.2（影响品牌形象）
- 包装问题：0.6（成本可控）

## 客服回复策略

### 回复原则
1. **先共情后解决**：不要一上来就解释，先认可用户感受
2. **个性化**：引用用户评论中的具体内容，避免模板感
3. **给出明确行动**：告诉用户你会做什么，而不是空话
4. **控制语气**：不卑不亢，不讨好也不过度道歉

### 回复模板结构
```
【共情】认可用户的具体不满
→ "理解您说的XX问题，确实让人不太愉快"

【解释/致歉】简短说明原因或致歉
→ "这次是因为XX原因导致的，我们很抱歉"

【解决方案】具体的处理措施
→ "我们已经为您XX，另外XX"

【预防承诺】避免再次发生的措施
→ "后续我们会XX，确保不再出现类似情况"

【感谢】结尾感谢反馈
→ "感谢您的反馈，帮我们发现了很多需要改进的地方"
```

### 禁用回复
- "亲，给个好评吧"
- "抱歉给您带来不便，祝您生活愉快"（万能结尾）
- 完全不回应用户具体问题的回复
