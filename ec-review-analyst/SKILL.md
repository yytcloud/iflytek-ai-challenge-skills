# 电商评论智能洞察分析师

## 触发条件

当用户出现以下任意场景时，使用本 Skill：
- 需要分析商品评论（好评/差评/中评）
- 提到"评论分析"、"用户反馈"、"差评处理"、"好评率"
- 需要从评论中提取用户痛点和需求
- 需要做竞品评论对比分析
- 需要生成客服回复建议或售后分类
- 提到"用户画像"、"消费洞察"、"转化优化"

## 核心能力

1. **细粒度情感分析**：不只是正/负/中，而是按维度（质量、物流、服务、包装、性价比）分别评分
2. **关键词智能提取**：从评论中自动挖掘高频提及词、情感词、场景词
3. **用户痛点聚类**：将分散的差评归因到具体问题类别
4. **竞品横向对比**：多款产品的评论数据结构化对比
5. **客服回复生成**：基于评论内容生成个性化、非模板化的客服回复

## 操作步骤

### 步骤 1：接收用户输入

用户需要提供**评论数据**。接受以下输入格式：

| 格式 | 示例 |
|---|---|
| 纯文本粘贴 | 直接粘贴多条评论，每条一行 |
| 结构化列表 | JSON / 表格形式的评论列表 |
| 页面截图描述 | 用户描述看到的评论内容 |

**每条评论最好包含**：评论文本 + 评分（如有）+ 时间（如有）

如果用户未提供评论，主动询问：
> "请提供需要分析的评论数据，可以直接粘贴评论文本。如果评论较多，建议包含评分信息以提高分析精度。"

### 步骤 2：加载分析模板

读取 `templates/analysis-framework.md`，获取电商评论分析的标准化维度框架。

### 步骤 3：预处理与清洗

对输入评论执行：
- 去重（完全相同的评论只保留一条）
- 去除无效内容（纯表情、无意义字符、"好评"三个字的空壳好评）
- 识别刷单嫌疑评论（内容高度相似、用词过于统一）
- 标记有效评论数量，在输出中说明清洗情况

### 步骤 4：多维度情感分析

对每条有效评论进行打分：

**情感极性**：
- `正面`（5星/好评）
- `中性`（3-4星）
- `负面`（1-2星/差评）

**维度评分**（每条评论可能涉及多个维度）：

| 维度 | 关注点 |
|---|---|
| 商品质量 | 材质、做工、功能、耐用性 |
| 外观设计 | 颜色、款式、颜值、尺寸 |
| 性价比 | 价格合理度、促销体验 |
| 物流体验 | 发货速度、包装完好、快递服务 |
| 客户服务 | 态度、响应速度、问题解决 |
| 使用体验 | 易用性、效果、场景适配 |

### 步骤 5：关键词与话题提取

提取以下类型的关键词：
- **高频词**：出现次数 ≥ 3 的名词/动词
- **情感词**：带有明确褒贬色彩的词（"好用"、"差劲"、"惊喜"、"失望"）
- **场景词**：用户提到使用场景的词（"送人"、"自用"、"出差"、"办公室"）
- **对比词**：提到竞品或替代品的词（"比xxx好"、"不如xxx"）

### 步骤 6：痛点聚类与根因分析

将所有负面评论按问题类型归类：

```
痛点类别 → 典型评论 → 出现频次 → 严重程度 → 改进建议
```

常见痛点类别参考：
- 质量缺陷（做工问题、功能故障）
- 描述不符（色差、尺码偏差、实际效果与宣传不符）
- 物流问题（破损、延迟、丢件）
- 服务问题（客服不回、退换货困难）
- 包装问题（简陋、挤压变形）

### 步骤 7：生成分析报告

输出标准格式的洞察报告：

```
一、数据概览
  - 总评论数 / 有效评论数 / 清洗率
  - 好中差比例

二、情感分布
  - 整体情感占比
  - 各维度情感雷达图（文字描述）

三、好评关键词 TOP10
四、差评关键词 TOP10

五、用户痛点排行
  - Top5 痛点 + 典型评论 + 频次

六、用户场景洞察
  - 主要使用场景
  - 购买决策因素

七、优化建议
  - 产品改进优先级
  - 营销话术调整建议
  - 客服话术优化方向

八、竞品对比（如用户提供）
  - 各维度得分对比表
  - 本品优势/劣势分析
```

### 步骤 8（可选）：生成客服回复

如果用户需要，为每条差评生成个性化回复，遵循 `templates/reply-template.md` 中的回复策略。

## 已知限制

1. 分析精度依赖评论数量——少于 20 条时，结论仅供参考
2. 刷单识别基于文本特征，无法 100% 准确判断
3. 情感分析基于语义理解，对高度反讽或隐晦表达可能识别不准
4. 不支持实时抓取评论，需用户提供数据
5. 竞品对比需要用户提供竞品的评论数据

## 最佳实践

- 评论越多，分析结论越可靠，建议 ≥ 50 条有效评论
- 提供评分信息可以提升维度分析的准确度
- 定期分析（如每月）可追踪用户满意度的变化趋势
- 结合销量数据一起看，可以识别"高销量但高差评"的隐患SKU
