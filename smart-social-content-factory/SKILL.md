# 智能社媒内容工厂

## 触发条件

当用户出现以下任意场景时，使用本 Skill：
- 需要为**小红书、抖音、微博、微信朋友圈、公众号**等平台写文案
- 提到"种草"、"发圈"、"写推文"、"做宣传"、"写脚本"、"写笔记"
- 有产品/服务/活动需要推广，需要适配社媒平台的内容
- 希望消除内容的"AI味"，让文案更自然、更像真人写的
- 需要标题优化、话题标签推荐、封面文案

## 核心能力

1. **多平台一键适配**：同一主题，生成 5 大主流平台的差异化文案
2. **AI 痕迹智能消除**：通过句式重构、口语化替换、情感注入，让内容像真人创作
3. **平台语言 DNA 匹配**：严格遵循各平台的语气规则、字数限制、格式惯例
4. **完整内容包输出**：正文 + 标题 + 话题标签 + 封面图文案建议

## 操作步骤

### 步骤 1：接收用户需求

收集以下信息（用户未提及的项，用默认值或主动询问）：

| 信息项 | 必填 | 默认值/说明 |
|---|---|---|
| 主题/产品 | 是 | — |
| 目标平台 | 否 | 全部 5 个平台 |
| 内容类型 | 否 | 种草推荐 |
| 风格偏好 | 否 | 亲切自然 |
| 核心卖点 | 否 | 从主题自动提炼 |
| 字数要求 | 否 | 按平台默认 |

**内容类型选项**：
- `种草推荐`：产品测评、好物分享
- `品牌宣传`：品牌故事、活动预告
- `知识科普`：干货分享、教程攻略
- `情感共鸣`：故事化内容、价值观输出
- `促销转化`：限时优惠、团购秒杀

**风格偏好选项**：
- `亲切自然`：像朋友聊天
- `专业权威`：专家背书感
- `幽默搞笑`：轻松玩梗
- `高级质感`：简约有调性
- `热血励志`：打鸡血风格

### 步骤 2：加载平台模板

根据目标平台，读取 `templates/platform-{平台}.md` 中的平台 DNA 规则，确保输出严格符合该平台的内容规范。

### 步骤 3：生成原始文案

基于主题和平台规则，生成初版文案。要求：
- 有具体细节（数字、场景、感受），避免空洞形容词
- 使用第一人称"我"视角，增强真实感
- 包含个人经历或观察，制造"真实体验"感
- 适当使用emoji，但不过度

### 步骤 4：AI 痕迹消除优化

对步骤 3 的文案执行以下改写，降低 AI detectability：

**句式层面**：
- 打破完美排比，改为长短句交错
- 将"首先...其次...最后..."改为口语化衔接（"说实话...然后呢...最关键的是..."）
- 插入个人口头禅、语气词（"啊"、"真的"、"说实话"、"其实吧"）

**词汇层面**：
- 替换过于正式的词汇为网络口语/生活用语
- 避免"值得一提的是"、"不难发现"、"总而言之"等模板化表达
- 使用不完美的比喻，避免过于精巧的修辞

**结构层面**：
- 偶尔加入无关紧要的细节（"那天刚好下雨"、"我在地铁上刷到的"）
- 允许逻辑上的轻微跳跃，模拟真实思维
- 结尾不刻意升华，可以自然收尾

**情感层面**：
- 加入具体情绪反应（"我当时都惊了"、"真的有点无语"）
- 表达轻微犹豫或不确定（"可能因人而异吧"、"至少我是这样觉得的"）

### 步骤 5：组装完整内容包

每个平台的输出必须包含：

```
【平台名称】

标题：（3个备选，标注主推）

正文：
（优化后的文案）

话题标签：（8-15个，按热度排序）

封面文案：（配合头图的简短文字，≤15字）

发布建议：（最佳发布时间、互动钩子设计）
```

### 步骤 6：多平台对比校准

如果用户需要多平台输出，确保：
- 各平台文案传达的核心信息一致
- 不存在平台间的内容冲突
- 根据平台调性合理调整表达深度和正式程度

## 平台输出规则

### 小红书
- 字数：300-800字
- 语气：姐妹分享、真实体验、细节控
- 结构：痛点引入 → 产品发现 → 使用体验 → 效果展示 → 购买建议
- 标签：#品类词 #场景词 #人群词 #功效词
- 禁用：过于硬广的语气、官方宣传腔

### 抖音
- 字数：100-300字（短视频脚本）
- 语气：节奏快、抓眼球、强情绪
- 结构：钩子（前3秒）→ 痛点共鸣 → 解决方案 → 行动号召
- 标签：#热点话题 #挑战赛标签 #品类标签
- 特殊：需标注"画面建议"和"BGM情绪"

### 微博
- 字数：100-500字
- 语气：时事感、话题性、可转发
- 结构：观点/爆料 → 论据/截图 → 互动提问 → 话题标签
- 标签：@相关账号 + #话题#
- 特殊：适合制造讨论和转发

### 朋友圈
- 字数：50-200字
- 语气：生活化、低调炫耀、真实感
- 结构：生活场景 → 自然带出 → 轻描淡写 → 引发好奇
- 标签：无需#
- 特殊：避免直接@品牌、避免硬广感

### 公众号
- 字数：800-2500字
- 语气：深度、有洞察、可信赖
- 结构：选题引入 → 背景分析 → 核心观点 → 案例支撑 → 总结升华
- 标签：文末放关键词
- 特殊：需要更强逻辑结构和信息密度

## 已知限制

1. 本 Skill 专注于**中文社媒文案**，暂不支持多语言输出
2. 需要用户明确"产品/主题"才能启动，纯创意发散场景效果有限
3. 封面文案建议基于文字逻辑，不包含视觉设计
4. 对于高度专业化的领域（医疗、金融投资），建议用户补充专业背景信息
5. 话题标签基于通用策略，非实时热榜数据

## 最佳实践提示

- 用户提供的信息越具体（如目标人群、价格区间、使用场景），输出质量越高
- 如果用户对某一平台输出不满意，可以指定平台要求"再改一版"
- 建议用户始终通过"AI痕迹消除"步骤，即使初版看起来已经不错
- 多平台同时输出时，建议用户优先选择 2-3 个核心平台深耕，而非全平台平均发力
